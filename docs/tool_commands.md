# Tool Commands Used

## OpenLane Initialization

```bash
docker
./flow.tcl -interactive
```

Purpose:

Launch the OpenLane interactive environment.

---

## Design Preparation

```bash
prep -design design_mux
```

Purpose:

Prepare the design for implementation and generate the working run directory.

---

## Synthesis

```bash
run_synthesis
```

Purpose:

Convert RTL Verilog into a gate-level netlist using Yosys.

---

## Floorplanning

```bash
run_floorplan
```

Purpose:

Generate die area, core area and initial physical planning information.

---

## Placement

```bash
run_placement
```

Purpose:

Place standard cells inside the floorplan.

---

## Routing

```bash
run_routing
```

Purpose:

Generate physical interconnections between cells.

---

## Magic Layout Inspection

```bash
magic
```

Purpose:

Open and inspect generated layout files.

---

## Report Analysis

Reports generated by OpenLane were inspected from:

```text
/openlane/designs/design_mux/runs/<run_directory>/reports/
```

Purpose:

Review timing, DRC, LVS and manufacturability results.
