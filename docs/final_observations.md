# Reference Flow vs AI-Assisted Flow

## Reference Repository Approach

The reference repository demonstrates a mixed-signal RTL-to-GDS flow for an analog 2:1 AMUX IP using OpenLane and the SKY130 PDK.

The repository contains manually prepared files, configuration settings, and implementation steps.

## AI-Assisted Approach

Instead of directly reproducing the repository structure, AI tools were used to:

* Analyze repository organization
* Explain mixed-signal concepts
* Identify required input files
* Understand OpenLane configuration
* Interpret reports and metrics
* Generate debugging strategies
* Assist documentation creation

## Verification Method

Every AI-generated explanation was manually verified through:

* Repository inspection
* OpenLane execution
* Report analysis
* Layout inspection
* Signoff verification

## Key Difference

The reference repository provides a completed implementation.

The AI-assisted flow focuses on understanding, generating, validating, and debugging the flow through structured prompts and engineering reasoning.

## Conclusion

The project demonstrates how AI can accelerate learning and debugging while still requiring manual verification and engineering judgment.
